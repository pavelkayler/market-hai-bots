# Project docs pack (drop into repo root)

Содержимое этого архива нужно скопировать в **корень репозитория** (рядом с `backend/` и `frontend/`).

Файлы:
- `PROJECT_SPEC.md` — единое ТЗ/спецификация текущего функционала (paper-only).
- `ROADMAP.md` — этапы стабилизации/регресса без расширения фич.
- `CODEX_RUNBOOK.md` — правила работы Codex: как брать задачи из чата, как делать PR/патчи, как тестировать.
- `PROMPT_ROLE_FOR_NEXT_CHAT.md` — *роль/инструкция для следующего чата* (не вставляй в чат, пока не скажешь **«Давай промт»**).
- `API_WS_CONTRACT.md` — контракт WS/REST (минимум, что UI и backend обязаны соблюдать).
- `TESTING_CHECKLIST.md` — что прогонять перед выдачей результата.

Принцип:
- **Функционал не расширяем** без явного запроса пользователя.
- Исправляем только связки/данные/кнопки/консистентность/регресс.
